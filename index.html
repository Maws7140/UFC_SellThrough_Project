Problem Statement

THE CHALLENGE OF PREDICTION

Domain: Sports analytics (UFC live events)

Business Problem: Predict event sell-through (tickets sold/capacity) to forecast attendance and revenue, driven by fight card, location, timing, and hype.

Data Blend: Internal (fight/event details, venue capacity) and External (betting odds, Reddit/Trends sentiment, fighter popularity graph).

Big-data approach: Use Spark to join, clean, and model millions of diverse records (history, ticket logs, social media, fighter networks) faster than a local device.

WHY SELL-THROUGH MATTERS

> Plan venues and ops with realistic attendance forecasts

> Set dynamic pricing and promotions to hit revenue targets

> Shape fight cards to maximize fan appeal and sell-through

Feature Engineering

[~] Rolling Fighter Statistics

Calculated fighter performance metrics over recent fights using window functions: win rate, finish rate, layoff time, average strikes landed, and takedown success rate.

[+] Graph-Based Features

GraphFrames to analyze fighter networks with PageRank scores, community detection, and network size metrics to capture fighter influence and matchup dynamics.

[$] Betting Features

Incorporated betting odds data: spread, implied probabilities, and competitiveness to reflect market expectations and fight competitiveness.

[#] External Data Features

Integrated Google Trends search interest and Reddit sentiment scores to capture pre-event buzz, fan engagement, and public sentiment.

[*] Event-Level Features

Card-level attributes: number of fights, title fights, rematches, average fighter win rate, event date features, and location indicators.

Challenges & Solutions

[!]

Missing Attendance Data

Only 85% of events had attendance data. Solved by fuzzy matching event names and date validation (±1 day) to maximize coverage.

[~]

Name Matching Across Sources

Event and fighter names varied across sources. Applied normalization (lowercase, special char removal) and fuzzy matching with date validation.

[%]

API Rate Limiting

Google Trends, Reddit, and betting sites imposed strict rate limits. Implemented request delays, retry logic with exponential backoff, and batch processing.

[#]

Data Format Variations

Handled inconsistent date and numeric formats across sources with robust parsing supporting multiple formats and currency symbols.

[?]

Historical Data Availability

External data like betting odds and social sentiment only available from 2010 onward. Adjusted analysis to account for limited features in older events.

[&]

Feature Engineering Complexity

Calculating rolling statistics for 4,400+ fighters over 8,400+ fights required scalable solutions. Leveraged PySpark window functions for distributed computation.

ALGORITHM OVERVIEW

Uses trained sell-through prediction model for fight card selection. Employs a greedy algorithm with 2-opt improvement for optimization. Considers fight importance, titles, weight classes, and fan appeal. Covers 10 different weight classes for diversity.
